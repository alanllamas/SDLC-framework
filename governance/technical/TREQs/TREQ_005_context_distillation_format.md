# Technical Requirement: TREQ_005 — Context Distillation Format

## 1. Intent & Scope
Structured format of context distillation maintained by
Orchestrator per TDR_003.

## 2. Technical Constraints
```yaml
distillation:
  session_id: "[ISO-8601 session start]"
  active_agent: "[Agent role]"
  active_phase: "[PHASE_1 | PHASE_2]"
  active_layer: "[BUSINESS | PRODUCT | ARCHITECTURE | TECHNICAL]"

  key_decisions:
    - decision: "[What was decided]"
      artifact: "[BDR/BREQ/ADR ID if applicable]"
      timestamp: "[ISO-8601]"

  open_questions:
    - question: "[What remains unresolved]"
      tquery: "[TQ_ID if applicable]"

  session_summary: >
    [2-3 sentence plain language summary]
```
* Updated after every authorized artifact or TQUERY resolution.
* session_summary regenerated by Orchestrator via LLM after each update.
* Raw conversation history retained in ~/.sdlc/ separately.

## 3. Verification & QA Criteria
* Distillation file present and valid YAML after every session.
* key_decisions count matches authorized artifacts in session.
* session_summary present and non-empty after first exchange.
