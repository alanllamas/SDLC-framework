# BREQ_012: The Unified TQUERY & Consultation Protocol

## 1. Core Objective
To establish a single, non-autonomous communication interface (`TQUERY`) that agents must use whenever they encounter an operational boundary, a git merge conflict, or an informational knowledge gap. This protocol guarantees that agents never guess, hallucinate, or make structural project decisions independently.

## 2. The Three TQUERY Trigger Types
Every `TQUERY` generated by an agent must explicitly categorize its block using one of the following three types:

* **`KNOWLEDGE_GAP`:** The agent lacks the raw business logic or context required to draft a proposal accurately.
* **`SCOPE_DRIFT`:** The active artifact is exceeding its complexity budget or drifting into adjacent subjects defined by BDR_011.
* **`CONFLICT_RESOLUTION`:** The Git Butler detects a merge block, branch drift, or version mismatch that cannot be cleanly combined.

## 3. The Programmatic TQUERY Template & Payload
To initiate a consultation or conflict resolution sequence, the originating agent must fetch the master template located at **`/governance/templates/TQUERY_template.yaml`**. 

The agent is strictly forbidden from constructing an ad-hoc or free-form text query. It must write a populated block into the active log channel conforming exactly to this schema:

```yaml
tquery:
  id: "TQ_XXX"                          # Unique sequential tracking ID
  trigger_type: "KNOWLEDGE_GAP | SCOPE_DRIFT | CONFLICT_RESOLUTION"
  originating_agent: "AGENT_ROLE"       # e.g., PM Orchestrator, Git Butler
  active_branch: "proposal/[Doc_ID]"    # The branch currently paused
  
  context_bounds:
    file_in_focus: "path/to/file.md"
    problem_statement: "Clear description of the gap or conflict in plain business terms."
  
  actionable_options:
    option_alpha: "Explicit description of Action A (and the architectural impact)."
    option_beta: "Explicit description of Action B (and the architectural impact)."
    option_custom: "Provide custom guidance/text to override."
  
  hitl_decision: null                   # Awaiting human input
